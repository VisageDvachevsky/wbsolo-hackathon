# wbsolo

Локальный рабочий репозиторий под хакатон по прогнозированию `target_1h`.

## Что в проекте

- `TZ.txt` — постановка задачи и описание метрики.
- `train_solo_track.parquet` — обучающая выборка.
- `test_solo_track.parquet` — тестовая выборка.
- `831419b5-3661-4980-94b3-081660040746.csv` — пример готового сабмита.
- `notebooks/` — EDA и быстрые эксперименты.
- `src/` — воспроизводимый код фичей, валидации и инференса.
- `reports/` — заметки по данным, метрике и результатам.
- `submissions/` — финальные выгрузки.

## План разведки

1. Проверить структуру данных и временную сетку.
2. Понять, как тест соотносится с train по `route_id` и времени.
3. Собрать локальную валидацию, близкую к лидерборду.
4. Построить простой воспроизводимый baseline.
5. Дальше итеративно улучшать признаки и модель.
